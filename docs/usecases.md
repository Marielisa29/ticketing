# Cas d'usage du système de ticketing

## Use cases principaux

1. **Créer un ticket** : Un utilisateur crée un nouveau ticket avec titre, description
2. **Assigner un ticket** : Un agent assigne un ticket à lui-même ou un autre agent
3. **Démarrer le traitement** : Un agent démarre le travail sur un ticket assigné
4. **Résoudre un ticket** : Un agent marque un ticket en cours de traitement comme résolu
5. **Clôturer un ticket** : Un administrateur clôture définitivement un ticket résolu
6. **Récupérer un ticket par ID** : Un utilisateur consulte le détail d'un ticket via son identifiant
7. **Lister les tickets** : Un utilisateur récupère la liste de tous les tickets, avec filtrage possible
8. **Réouvrir un ticket** : Un agent réouvre un ticket résolu si le problème persiste
